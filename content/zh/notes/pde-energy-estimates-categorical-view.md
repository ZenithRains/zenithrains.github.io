我曾经想过 PDE 能量估计的问题。能量估计相当于一种粗粒化，把 PDE 变成一个 ODE 问题。做能量估计的时候，经常会出现一种情况。每次写 Young 不等式的时候，都可以说取一个足够小的 $\varepsilon$。这些 $\varepsilon$ 不能各取各的。前面某个中间估计可能要给后面两个地方一起用。处理同一个非线性项，也可能有好几条路线。每一小步单独看都没什么问题，放在一起就可能打架。这里面有复用，有分叉，有不同路线的比较，还有局部参数的全局相容性。它应该有一个自己的数学结构。那么还可以注意到，有时候人们为了封闭估计，还会使用新的不等式。大概一年以前，我在看泛函分析的时候，发现了函数的零点有个定理，看着特别眼熟。类似 Hilbert 零点定理。这一次在这里又看到了相似的结构。当然这有个学名叫做 Galois 对应。然后我就想到了范畴论。也就是 PDE 能量估计的动作理论上可以被范畴论刻画。

先看一个普通的能量不等式。在时间区间 $t\in(0,\mathcal T)$ 上，假设

\[
\frac{1}{2}E'(t)+\lambda\mathbin{\cdot}D(t)
\leq \sum_{\nu=1}^{m}N_\nu(t).
\]

这里 $E$ 是能量，$D=(D_1,\ldots,D_d)$ 是若干耗散量，$\lambda\in(0,\infty)^d$ 是手里的耗散预算，$N_\nu$ 是需要处理的非线性项。表面上看，接下来就是逐个估计 $N_\nu$。实际写起来完全不是这么线性的。一个 $N_\nu$ 先用 Hölder，再用插值，然后用 Young。另一个 $N_\mu$ 可能会调用刚才得到的中间估计。同一个目标也可能有两种处理办法。一种省导数，但是常数很差。另一种多用一点正则性，但是耗散负载更小。所以做 PDE 的人的大脑里面真正存在的东西，其实是一张有限的估计图。图上的方框是一条 Hölder、Sobolev、交换子、迹、Hardy、Young 或者问题特有的不等式。连线表示一个估计怎么进入下一步。一个结论可以分叉出去，被后面重复使用。两条不同的路线也可以到达同一个目标。

我一开始只想写 Galois 对偶。写着写着其实有一个问题。如果只记录“这个估计能不能推出”，那就把证明是怎么走过去的全忘了。可是 PDE 分析者恰恰很关心路线。于是范畴应该先出现。Galois 对偶是把路线忘掉以后留下来的序结构。把这些搞 PDE 能量封闭的人平时做的事情一项一项写出来，就得到下面这张表。

| PDE 分析行为 | 范畴论表示 |
|---|---|
| 一个完全指定的估计 | 一个生成对象；若干可用估计组成一个上下文 |
| 用若干估计推出另一个估计 | 一个多输入的基本方框 |
| 连续使用多个不等式 | 态射的复合 |
| 同时进行彼此独立的估计 | 张量积 |
| 重复使用一个中间估计 | 一条线的分叉 |
| 两条不同的估计路线 | 平行态射 |
| 把一段已有计算抽取成引理 | 给已有复合加入一个生成元，并声明二者相等 |
| 减少中间估计 | 在固定表示下寻找较短的因子分解或证明图 |
| 只记录指数和导数阶数 | 特征函子 |
| 指数计算可行，但具体证明不存在 | 抽象态射没有具体提升 |
| 选择 Young 或插值参数 | 在受约束的提升空间中选择一点 |
| 每一步都能实现，但整体不能吸收 | 提升存在，而可行子空间为空 |
| 把一种方法移植到另一类 PDE | 分析迁移函子 |
| 系统地比较两种估计方案 | 满足相容条件的自然变换 |

当然，表里的“一个估计”必须写完整，只写一个 $\|u\|_p\leq C\|u\|_{H^s}$ 肯定不行。在哪个区域？什么时间区间？什么边界条件？解属于什么类？参数怎么取？常数依赖哪些量？这些内容都写进去以后，这条估计才有确定的输入和输出。否则对象都还没有定义清楚，后面谈 composition 就很虚。然后连续使用 Hölder、插值和 Young，就是复合。两段互不相干的估计可以并排进行，就是张量积。一个已经得到的能量界要送到两个非线性项里面，那条线就发生了分叉。两条路线到达同一个目标，就是平行态射。这样一来，我们不光知道目标能推出，还知道它是怎么被推出的。表里面还有“减少中间估计”这一行。所谓更短，只能是在当前选定的生成元和画图方式下更短。换一套基本引理，整张图的长度就会变。把一段已有计算命名成一个引理也是一样。我们需要声明这个新方框和原来的复合是同一个推导，不然就是偷偷加了一条新规则。

PDE 人做先验估计的时候，经常会先只算指数。先不管边界，不管端点，不管常数。就看导数阶数对不对，时间和空间的 Lebesgue 指数能不能配起来，能量和耗散各占多少次方。这个办法很好用。因为一条路线如果连指数都配不出来，就没必要继续浪费时间了。从范畴论的角度说，这一步相当于忘掉大部分分析数据，只保留一些特征。只要这个遗忘尊重判断、规则和输入输出类型，它就可以写成一个特征函子。最后得到一张抽象的证明图。可是指数算通以后，还要把刚才扔掉的东西一个个放回去。边界条件放回去。端点限制放回去。正则性要求和常数依赖也放回去。这个过程可以看成一个提升问题。这样有一种很常见的失败可以说得很清楚：抽象图存在，但是这张图提不起来。可能某一步需要的端点估计根本不成立。可能两个地方对正则性的要求互相冲突。也可能常数无法在所需参数上一致。以前我可能只会说“这条路线算不通”。现在可以说得更准确一点。指数草图已经通过了，具体提升失败了。然后还有 Young 参数的问题。论文里面写一句“取 $\varepsilon$ 足够小”很轻松。写五次以后，事情就不轻松了。这五个参数最后要一起结账。每一步返回到耗散项上的负载，都要放在一起和 $\lambda$ 比较。

所以我们现在可以把闭合分成以下三层：

1. 先看有没有一张抽象的证明图；
2. 再看这张图能不能提升成满足具体分析条件的证明；
3. 最后把所有参数放在一起，检查余项和耗散预算。

第二层过了，第三层仍然可能过不了。用集合语言说，就是提升空间里面有点，但是满足最终闭合条件的可行子空间为空。有可能每一步都能做，整体就是不能吸收。

现在把具体路线忘掉，只问一个估计能不能从现有规则推出。所有能够推出的估计组成一个可达闭包。然后再看所有对规则稳定、取值于真假的模型。它们会给出相应的句法障碍。判断和这些稳定模型之间形成一个 Galois 对偶。当然这个障碍只说明目标在当前规则系统里面不可达。它不一定来自某个真实的 PDE 解，也不是解析反例。要得到 PDE 反例，还得真的回到方程和解上面去构造。当所有相应的稳定模型都被放进来以后，Galois 对偶完整描述的是句法可达性。还有一个 Galois 连接出现在特征遗忘这件事里面。具体判断映到特征判断，两个幂集之间自然会有像映射和逆像映射。这两个映射形成 Galois 连接。如果抽象域只收录实际出现的特征判断，就进一步得到 Galois 插入。

通过大模型搜索资料，我发现这就和 [Cousot--Cousot 的抽象解释](https://cs.nyu.edu/~pcousot/COUSOTpapers/POPL77.shtml)接上了。他们本来是在讨论程序分析：怎么把具体语义送到一个比较粗的抽象域里面，同时保证抽象计算是可靠的。太乐了，PDE 分析者先算指数、再补具体条件，就是在干类似的事情。他们后来的 [Galois connection calculus](https://cs.nyu.edu/~pcousot/COUSOTpapers/POPL14.shtml)又把 Galois 连接本身做成了一套系统演算。这里还有一个区别。集合层面的闭包会把证明路线忘掉。某个特征目标可能确实能由某一条具体路线到达，但是我们手里指定的那张抽象图仍然提不起来。所以我还是想保留前面的证明范畴。它比一个单纯的闭包多记了一些东西。

接下来就是我觉得最好玩的地方了。如果我们找到一条新的不等式，相当于给证明系统增加了一个新的基本方框。只要旧规则和旧关系没有被改掉，原来的路线就都还在。新的方框可以接在旧规则前面，也可以接在后面。于是以前拼不出来的证明图，现在可能突然拼出来了。从 Galois 对偶的另一侧看，原来的一些句法障碍也会失效。加一条新的不等式，就是给证明图增加一种新的走法。以前走不通的地方，现在可能通了。新不等式也可能是冗余的。那就什么都没有改变。可是如果它正好补上了图里缺的那一块，整个论证就会突然打开。这就是看 PDE 论文，某位作者突然用了一个新的估计或者新的不等式闭合了整个不等式。

有了这张图以后，还可以继续问很多问题。比如我们经常说：“这个方法在另一个方程里面也能用。”真的搬过去，需要一起搬前提、基本不等式、边界条件、参数限制、常数依赖、余项类别和耗散预算。如果整套东西都有相容的对应关系，那么这个迁移可以写成一个函子。如果有两种迁移方案，还可以问它们能不能逐项相容地比较。这个时候就会出现自然变换。当然，自然变换只能告诉我们比较是相容的。哪种方法更强，哪种路线更短，谁消耗的耗散更少，还得真的去算。

现在也不能把这件事说得太大。目前这个模型没有给出新的 PDE 定理，也不会自动替我们发明估计。它现在做成的事情，是把 PDE 分析者遇到的问题符号化。一个方案失败了，我们可以继续问：根本没有抽象路线？还是路线有了但是提不起来？是不是几个局部参数互相打架？还是最后就差一点耗散预算？不过说真的这已经很好玩了。具体的不等式当然还得一条一条证明。但是我们终于可以把这些不等式怎么接起来、在哪里分叉、在哪里失败，作为一个数学对象来研究。接下来也许可以搜索图里面缺失的规则，定位最先失败的子图，或者比较两种闭合方案到底在哪一步走向了不同的方向。
